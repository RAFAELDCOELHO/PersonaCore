# Phase 8 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase surfaces (this is not a web app):** Phase 8's "UI" is three artifacts —
(1) the offline Gradio 5.50.0 `gr.ChatInterface` demo (`scripts/demo_app.py`, DEMO-01),
(2) the executed `demo.ipynb` figures (DEMO-03), and
(3) the README hero block (GIF + numbers, DOC-01/D-04).
There is no React/Next.js/Vite stack, so the shadcn gate is **not applicable**. The governing
principle for the demo surface: **Gradio default theme, unmodified** — the offline guarantee
(wheel-bundled `LocalFont("IBM Plex Sans")`, no CDN calls) was verified in 08-RESEARCH against
the *default* theme only. Any custom theme/font/CSS voids that verification and is prohibited
this phase.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (no shadcn — Python/Gradio stack) |
| Preset | not applicable |
| Component library | Gradio 5.50.0 built-ins only (`ChatInterface`, `Slider`) — theme-supplied, unmodified `[source: 08-RESEARCH, verified from wheel]` |
| Icon library | none (Gradio built-in icons only: submit arrow, stop, retry/undo/clear) |
| Font | Gradio default: IBM Plex Sans (body) / IBM Plex Mono (code) — `LocalFont`, wheel-bundled woff2, REQUIRED for offline `[source: 08-RESEARCH §Gradio facts]` |

**Hard rule:** no `theme=` argument, no `css=` argument, no `gr.themes.*` customization, no
GoogleFont. The default theme is the only offline-verified configuration (08-RESEARCH Pitfall 5
context). Notebook/README typography is renderer-supplied (GitHub/Jupyter defaults) — no
embedded HTML styling in markdown cells.

## Spacing Scale

Declared values (must be multiples of 4). Applies to authored layout decisions (matplotlib
figure padding, README asset sizing). Gradio surface spacing is theme-supplied and unmodified.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | (theme-supplied — Gradio inline gaps; never overridden) |
| sm | 8px | Matplotlib tight_layout pad equivalents; gap between README badges/numbers |
| md | 16px | Default spacing between README sections' elements |
| lg | 24px | (theme-supplied — Gradio block padding; never overridden) |
| xl | 32px | Vertical rhythm between README hero GIF and results block |
| 2xl | 48px | Major REPORT.md section breaks (rendered via `---` rules + heading gaps) |
| 3xl | 64px | not used this phase |

Exceptions:
- **GIF width 720px** (asset dimension, not a spacing token) — fixed by the ffmpeg recipe
  `scale=720:-1` `[source: 08-RESEARCH Pattern 5]`.
- **Matplotlib figures sized in inches:** `figsize=(8, 4.5)` at `dpi=100` → 800×450 px rendered,
  keeping the committed notebook under the ~5 MB bloat ceiling (08-RESEARCH Pitfall 7).

## Typography

Scale applies to authored surfaces. Gradio and GitHub renderers supply conforming defaults;
the rule is **no size/weight overrides anywhere** — exactly these four sizes and two weights
appear in anything we control.

| Role | Size | Weight | Line Height | Where |
|------|------|--------|-------------|-------|
| Body | 16px | 400 | 1.5 | Gradio chat text + description (theme default); README/REPORT body (GitHub default) |
| Label | 14px | 400 | 1.5 | Slider labels in the controls accordion (theme default); figure captions in notebook markdown |
| Heading | 20px | 600 | 1.2 | README/REPORT `##` sections; notebook section headings |
| Display | 28px | 600 | 1.2 | Gradio `title` (theme h1); README `#` title |

**Matplotlib text contract (the one place we set sizes directly):** axis labels 11pt, figure
title 12pt, legend/tick labels 10pt — matplotlib defaults, NOT overridden via rcParams. One
deliberate setting only: `dpi=100` (Pitfall 7). Weights in plots: regular only.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #FFFFFF | Gradio light-mode body background; notebook figure backgrounds; GitHub README canvas |
| Secondary (30%) | #F9FAFB (neutral-50) | Gradio chat panel, accordion, input surfaces (theme-supplied) |
| Accent (10%) | #FF7C00 (Gradio orange) | Theme-supplied accent — see reserved list below |
| Destructive | none | No destructive actions in this phase (see Copywriting) |

Accent reserved for: **the Gradio submit (send) button and slider active track/handle only** —
both theme-supplied. Nothing custom is ever painted accent: no orange text in README, no orange
matplotlib lines styled to "match the brand."

**Plot color contract (matplotlib default cycle, never customized):**
- Training-curve figure: train loss = `C0` (#1f77b4), val loss = `C1` (#ff7f0e) — same mapping
  in every figure that shows both.
- Ablation cohort figure: 4 runs on the default cycle in this fixed order — baseline = `C0`,
  no_tie = `C1`, no_pos = `C2`, depth_cut = `C3` — with a legend; consistent across all
  ablation plots.
- LR curve (if plotted): `C0` alone on its own axes.

**Capture contract:** the README GIF is recorded with the demo in **light mode** so it matches
the white-background notebook figures embedded nearby (Gradio follows system preference —
force light for the capture session).

## Copywriting Contract

Honesty bar (locked D-02, 08-RESEARCH Pitfall 9): the demo, README, and notebook must never
claim chat tuning or weight-memory for M1. Framing is **story completion**, not assistant chat.

| Element | Copy |
|---------|------|
| Primary CTA (README quickstart) | "Run the demo" → `python scripts/demo_app.py` (after `pip install -e ".[cpu,demo]"` + checkpoint download/regeneration steps) |
| Demo CTA (submit) | Gradio default submit button — unmodified (icon, theme-supplied) |
| Demo title (`title=`) | `PersonaCore — TinyStories story completion (13.9M params, fully on-device)` |
| Demo description (`description=`) | `A from-scratch GPT running on your CPU — no internet, no API, no database. Type a story opening and the model continues it. Each message starts a fresh story (previous turns are not fed back in). This is the Milestone-1 base model: no chat tuning, no personalization yet — that's Milestone 2.` |
| Textbox placeholder | `Type a story opening…` |
| Empty state | `examples=` with exactly 3 clickable story openings: `Once upon a time, there was a little dog named Max.` / `One day, a little girl named Lily found a shiny stone.` / `Tom and his cat went to the park to play.` (keep `cache_examples` default False — caching would run the model at launch) |
| Error state — missing checkpoint (launcher startup) | `checkpoints/model_slim.pt not found. Either download the release asset (see README quickstart) or regenerate it from a local best.pt: python scripts/export_slim.py` — raised as `FileNotFoundError` with this message before any Gradio import side effects |
| Error state — generation failure | Gradio default error toast (theme-supplied); no custom error UI. The existing `max_new_tokens ≤ 4096` guard raises before the loop; slider bounds make it unreachable from the UI |
| Destructive confirmation | none required — ChatInterface's built-in clear/retry/undo act on display-only history (fresh story per message; no state is stored), so no confirmation flow exists this phase |

**Control labels (exact strings + ranges, from the verified 08-RESEARCH Pattern 1):**

| Control | Label | Range / Default |
|---------|-------|-----------------|
| `gr.Slider` | `Temperature` | 0.1–1.5, default 0.8, step 0.05 |
| `gr.Slider` | `Top-k (0 = disabled)` | 0–200, default 50, step 1 |
| `gr.Slider` | `Max new tokens` | 16–1024, default 400, step 16 (under the 4096 DoS cap) |

**Discretion resolutions (delegated by 08-CONTEXT "Claude's Discretion"):**
- **Top-p is NOT exposed in the demo.** The locked requirement is temperature/top-k; top-p
  belongs to the notebook's D-07 settings tour (greedy vs temperature vs top-k/top-p), where
  its effect is shown side-by-side. Keeping the accordion to 3 controls keeps the demo honest
  and the GIF clean.
- **Multi-turn = fresh story per message.** History is never concatenated into the prompt
  (08-RESEARCH anti-pattern: zero dialogue tuning → concatenation produces incoherence and
  overclaims chat ability). The description states this explicitly.
- **Chat metaphor is reframed as story completion** in the title, description, placeholder,
  and examples — the ChatInterface widget is just the streaming vehicle.

## Interaction Contract

| Interaction | Contract |
|-------------|----------|
| Streaming | Callback accumulates `generate_text` deltas and yields the cumulative string (`acc += delta; yield acc`) — the chat bubble grows; it never "flickers" lone fragments (08-RESEARCH Pitfall 1) |
| Latency expectation | First visible chunk in milliseconds; a full ~200-token story streams in ~2 s (~95–105 tok/s CPU, measured). No spinner/skeleton needed — streaming IS the loading state |
| Stop | Gradio built-in stop button during generation (theme-supplied; no custom handling) |
| EOS | Generation ends naturally on EOS via the Phase-6 wrapper; no raw separator token is ever shown |
| Launch | `share=False`, localhost:7860; `GRADIO_ANALYTICS_ENABLED=False` set before `import gradio` + `analytics_enabled=False` passed — zero outbound network calls; UI must load and stream with Wi-Fi off |
| History UX | `type="messages"`; `save_history` stays default False (no browser localStorage) |
