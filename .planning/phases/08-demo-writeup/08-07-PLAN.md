<objective>
Close the one blocking verification gap (CR-01): the flagship Gradio demo crashes
mid-generation at slider settings its own UI offers, because the model samples over the full
8192-id table while the frozen production tokenizer decodes only 547 live ids (256 bytes +
283 learned merges + 8 specials; 7645 dead ids), and `BPETokenizer.decode` raises plain
`ValueError` for any dead id while the streaming wrapper catches only `UnicodeDecodeError`.
Measured crash probability at the slider extremes (temp 1.5, top-k disabled): ~29% per
400-token generation.

The locked fix (08-VERIFICATION `missing` list — the contract): mask tokenizer-undecodable
ids out of the logits BEFORE sampling, via an optional `forbid_ids` mask threaded through
`next_token`/`generate`, built in `build_demo` from the live vocab + specials. Do NOT fix by
catching `ValueError` — that would silently truncate output and swallow genuine strict-decode
defects. Pin the contract with regression tests. Opportunistically fix WR-02 (`export_slim`
opaque TypeError on `val_loss=None`).

Purpose: DEMO-01 — the live on-device proof must not crash at settings its own controls offer.
Output: forbid-mask sampling path (tested), demo wiring, `tests/test_forbid_ids.py`,
None-safe `export_slim`.
