Implement `class BPETokenizer` holding `merges: dict[tuple[int,int],int]`, `vocab: dict[int,bytes]`, `pattern: str`, `special_tokens: dict[str,int]`, `eos_id: int`, `vocab_size: int`:

    - `train(self, text, vocab_size=8192)`: pre-split `text` via `_COMPILED.findall`, encode each chunk to UTF-8 byte-id lists, then loop assigning merge ids starting at **256** (bytes occupy 0-255; specials are top-pinned at 8184+, so merges never collide). Number of merges = `vocab_size - 256 - len(special_tokens)` (= ~7928 at 8192). Each step: aggregate `get_stats` over all chunks, pick the pair with a **total deterministic tie-break** `pair = max(stats, key=lambda p: (stats[p], p))` (Pitfall 1 — never bare `stats.get`), assign the next id, `merge` it across all chunks, record `merges[pair]=idx`. Build `vocab` (0-255 single bytes ∪ merge-id → concatenated bytes). Return `self`.
    - `_encode_chunk(self, text_bytes)`: lowest-rank-first replay (RESEARCH Pattern 3) — `while len(ids)>=2: pair = min(stats, key=lambda p: self.merges.get(p, inf)); if pair not in merges: break; merge`.
    - `_encode_ordinary(self, text)`: `_COMPILED.findall` → per chunk `_encode_chunk(chunk.encode("utf-8"))`, concatenated. NEVER strip/normalize (Pitfall 6).
    - `encode(self, text, allowed_special="all")`: split FIRST on a capturing regex alternation of the special-token strings sorted by descending length (Pattern 4 / Pitfall 3); special parts emit their reserved id atomically; ordinary parts go through `_encode_ordinary`. With `allowed_special="all"` all 8 specials are recognized.
    - `decode(self, ids)`: join `vocab[id]` bytes for merge/byte ids, map special ids back to their literal text bytes via an inverse-special map, raise on unknown id, then `.decode("utf-8")` (Pattern 5; `errors="replace"` only as a never-triggered safety net since byte-level guarantees no `<unk>`).
    - `@classmethod frozen(cls, *, pattern, merges, special_tokens, eos_id, vocab_size)`: rebuild a ready-to-use tokenizer from a freeze dict (mirrors `load_checkpoint`), reconstructing `vocab` from `merges` so `decode` works. This is the constructor `io.from_json` (Plan 03) calls.
