<objective>
Build the thin text wrapper — the `str -> str` surface the Phase-8 Gradio demo will call — on top of
the id-space core from 06-02. `generate_text(model, tokenizer, prompt, ...)` prepends `[eos_id]` to
match the trained document-start register (D-03), drives the core, decodes the whole running buffer
each step and yields only the new string suffix to avoid byte-level-BPE mojibake (D-06), strips the
prompt, and returns only the continuation (D-02). It is a generator of new-suffix strings with a thin
`"".join(...)` convenience for non-streaming callers.

Purpose: Complete the "one function, three consumers" layering — the id-space core serves tests/eval,
this wrapper serves the demo — behind a single decode path. D-05 (stop-without-yield-EOS) + D-06
(running-buffer delta) together guarantee clean streamed text with no `<|endoftext|>` artifact.

Output: `src/personacore/generation/text.py`, updated `generation/__init__.py` barrel, and
`tests/test_generation_text.py` (CPU-only wrapper tests using a stub tokenizer + tiny model — never
best.pt / the real tokenizer.json).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user prompt -> generate_text | The prompt is the one externally-influenced input. It is passed to the byte-level tokenizer.encode (cannot produce out-of-range ids) and never executed, eval'd, or used in a query. No network, no auth, no DB. |
| caller -> max_new_tokens | An integer loop bound; capped to prevent an unbounded generation loop. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-04 | Denial of Service | generate_text(max_new_tokens) | mitigate | Reject max_new_tokens > max_new_tokens_cap (default 4096) or <= 0 with a ValueError before the loop; the core loop is range-bounded so it always terminates (V5). Verified by test_max_new_tokens_cap. |
| T-06-05 | Information Disclosure | streamed text containing the document separator | mitigate | D-05 stops the core without yielding EOS and D-06 running-buffer decode never surfaces a partial/raw separator; verified by test_no_raw_eos_in_output. |
| T-06-06 | Tampering | strict-decode crash on adversarial/partial bytes | mitigate | Running-buffer delta decode (D-06) only ever decodes a complete cumulative buffer; strict decode surfaces genuine defects rather than silently corrupting. Verified by test_running_buffer_no_mojibake. |

No high-severity threats. Surface is a local pure-compute text wrapper: a local prompt string + a trusted in-process model, no network, no auth, no DB, no new serialization (the frozen tokenizer is read-only JSON, loaded only by callers, not in tests). No package installs — Package Legitimacy Gate not applicable.
</threat_model>
